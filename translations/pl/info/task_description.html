Dodatnia liczba całkowita n jest liczbą perfect power, jeśli można ją wyrazić jako potęgę be dla pewnych dwóch liczb całkowitych b i e, które są większe od jeden. (Każda dodatnia liczba całkowita n może być zawsze wyrażona jako trywialna potęga n1, więc nie dbamy o te) Na przykład, liczby całkowite 32, 125 i 441 są potęgami doskonałymi, ponieważ są równe odpowiednio 25, 53 i 212.

Funkcja ta powinna określić, czy dodatnia liczba całkowita n jest potęgą doskonałą. Pana funkcja musi w jakiś sposób iterować przez wystarczającą liczbę możliwych kombinacji b i e, które mogą działać, zwracając {% if is_js %}t{% else %}T{% endif %}rue natychmiast po znalezieniu kombinacji b i e, które spełniają warunek be == n i zwraca {% if is_js %}f{% else %}F{% endif %}alse gdy wszystkie odpowiednie możliwości dla b i e zostały wypróbowane i uznane za nieodpowiednie.

Ponieważ n może być dość duże, funkcja nie powinna sprawdzać zbyt wielu kombinacji ponad te, które są zarówno konieczne, jak i wystarczające do wiarygodnego określenia odpowiedzi. Osiągnięcie tej wydajności jest głównym punktem edukacyjnym tego problemu.

W związku z misją, mogą być Państwo zainteresowani Catalan’s conjecture, obecnie udowodnionym twierdzeniem matematycznym, które mówi, że po szczególnym przypadku dwóch kolejnych potęg doskonałych 8 i 9, ilekroć dodatnia liczba całkowita n jest potęgą doskonałą, n – 1 nigdy nie jest potęgą doskonałą. Przykładowo, nie musimy przebijać się przez wszystkie potencjalne sposoby wyrażenia liczby jako potęgi liczby całkowitej, aby od razu wiedzieć, że 1234567890-1 nie jest potęgą doskonałą. Ilustruje to również powszechną asymetrię między wykonywaniem obliczeń w przeciwnych kierunkach. Biorąc pod uwagę jakąś dużą liczbę całkowitą, taką jak 4922235242952026704037113243122008064, ale nie wzór, który ją pierwotnie wygenerował, nie jest łatwo stwierdzić, czy ta liczba całkowita jest potęgą doskonałą, czy też potęgą doskonałą plus minus jeden.
